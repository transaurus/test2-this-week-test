## 什麼是 Docusaurus？

Meta 有大量內外部專案需持續編寫文件。撰寫文件已具挑戰性，若每次還需自訂網站，更會耗費大量時間。

2017 年時他們開始為每個專案複製貼上 Jekyll 模板，但維護迅速變得困難。

2017 年，他們決定建立 Docusaurus——基於 Node.js 和 React 的靜態網站生成器，可「隱藏」技術堆疊並讓你**專注內容**：撰寫 **Markdown** 檔案。類似 Create-React-App，但專為內容網站設計：文件、部落格、登陸頁面...

Docusaurus v1 迅速普及，被 React-Native、Jest、Babel、Prettier 等專案採用...

## 邁向 Docusaurus v2

v1 發布後，Meta 意識到其限制：

- 僅在伺服器端使用 React：沒有水合（hydration）也沒有單頁應用程式（SPA）導航
- 儲存庫結構相當龐大單一，難以維護和測試
- 擴展性不足，缺乏插件系統
- 主題功能相對有限，且 v1 網站外觀高度相似

2018 年，Meta 決定**從頭重建 Docusaurus**，採用全新的架構：

- 在客戶端和伺服器端都使用 React（類似 Next.js、Remix、Gatsby...）
- 插件系統：讓社群能貢獻實用功能（OpenAPI 整合、本地搜尋引擎、Sass 支援等）
- 更靈活的主題系統：可覆寫主題中的任何 React 元件
- MDX（Markdown + React）讓文件具備互動性

Docusaurus 2.0 經歷了 **4 年** 🤪 的 alpha/beta 階段，終於發布**首個穩定版本**！延遲發布是因先前難以符合語義化版本規範。

如今 Docusaurus v2 具備眾多功能，難以完整列舉：文件系統、部落格、版本控制、主題、深色模式、國際化（i18n）、無障礙（a11y）、SEO、TypeScript、PWA、Algolia DocSearch、MDX...
