## ⚛️ React {#react}

[![tRPC 10](/emails/issues/125/trpc10.jpg)](https://trpc.io/blog/announcing-trpc-10)

**[tRPC 10](https://trpc.io/blog/announcing-trpc-10)**

tRPC 的重大重構顯著提升開發者體驗(DX)和 IDE 支援。這個實現類型安全後端調用的函式庫日益受到歡迎，得益於整合套件(Next.js、React-Query...)，能無縫融入 React 生態系。我特別喜歡其標語「快速行動，不破壞任何」😄

為理解核心理念，推薦閱讀[為何我們棄用 GraphQL 改用 tRPC](https://echobind.com/post/why-we-ditched-graphql-for-trpc)。Alex 解釋了他們在 React 環境的[Bison](https://github.com/echobind/bisonapp)入門專案中替換 GraphQL 的原因。tRPC 大幅減少樣板程式碼、TypeScript 程式碼生成和 JS 套件體積，同時提升 DX、IDE 支援並簡化快取管理。處理過度獲取(over-fetching)的方式不同：應積極創建新程序。tRPC 特別適合全端 TypeScript Node.js 應用與同地協作的前後端程式碼(如 Next.js)。GraphQL 仍是建立公開 API 的優選方案，尤其適用於非 JavaScript 使用者且需程式碼生成的場景(OpenAPI 也是此類需求的好選擇)。

藉此機會我也要介紹 [T3-App](https://create.t3.gg/)，這是一個基於 tRPC、Prisma、Tailwind 的 Next.js 端到端類型安全堆疊。它極大地推動了 tRPC 今年的普及，現在已擁有官方文件網站。其 [tRPC 文件](https://create.t3.gg/en/usage/trpc) 清楚說明了相較於傳統 Next.js Route API 的優勢。另可參考 [create-t3-turbo](https://github.com/t3-oss/create-t3-turbo)：一個採用 Turborepo 的單體倉庫，包含 Next.js + Expo，均使用 tRPC。

**[Remix 與網頁開發的替代時間線](https://blog.jim-nielsen.com/2022/remix-alternate-timeline-of-web-dev/)**

Jim 闡述 Remix 如何透過回溯思考，重新設計前端應用模型。在 Remix 中，漸進增強功能無需像其他框架般分寫客戶端/伺服器雙重邏輯，因為單一路由即可從相同程式碼返回 JSON 與 HTML。
