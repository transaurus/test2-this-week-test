## React

[**陳述式 vs. 表達式**](https://www.joshwcomeau.com/javascript/statements-vs-expressions/?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Josh Comeau 正在籌備他的下一個 React 課程。他建議回歸 JavaScript 基礎，從而更好地理解 JSX 的運作原理（因 JSX 僅適用於表達式）。同主題下，我樂見 [do-expressions](https://github.com/tc39/proposal-do-expressions?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter) 提案能有所進展 😇。

[**如何在 Storybook 中建置關聯元件**](https://storybook.js.org/blog/how-to-build-connected-components-in-storybook/?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Storybook 官方文章詳解裝飾器(decorators)的各種應用：添加佈局、全局提供者(global providers)、模擬 Redux 儲存庫或請求... 這能協助在 Storybook 中加入因上下文依賴而無法渲染的「關聯」元件。

[**我如何評估 NPM 套件市佔率（以及 Redux 與其他函式庫的比較）**](https://blog.isquaredsoftware.com/2022/07/npm-package-market-share-estimates/?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Mark Erikson (Redux) 解釋他如何基於各種可靠性不一的工具分析 npm 套件使用趨勢。下載統計數據常與 CI 建置相關。他以 React 狀態管理器為例，現估計 Redux/React 使用率為 33%（先前為 45-50%）。

[**React 中的無動作與無狀態 Reducers**](https://julesblom.com/writing/actionless-and-stateless-reducers?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

useReducer() 不僅限於複雜情境：也能用於極簡單的狀態機，例如將布林值從 false 切換為 true。
