## React 生態圈

[Fresh 1.0](https://deno.com/blog/fresh-is-stable?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

由 Deno 官方支持的 Preact 框架現已發布 v1 正式版。Fresh 融合了 Remix 和 Astro 的特點：預設無 JS、漸進增強、採用島嶼架構的多頁面應用。無需建置步驟：可直接將 TypeScript/Deno 應用部署到邊緣環境，僅需數秒。

[我們如何建構 Hydrogen](https://shopify.engineering/how-we-built-hydrogen?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Shopify 的 React 元框架 Hydrogen 剛發布 v1.0，是最早採用 React Server Components 的框架之一。本文深入分享框架開發歷程，詳述各項技術決策：React Server Components、Vite、Tailwind、預加載/瀑布流問題、GraphQL 片段...

[Next.js 12.2](https://nextjs.org/blog/next-12-2?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Next.js 最新版本。核心主題是逐步採用邊緣運算和標準化 Web API：包含中介層(middleware)、API 路由和 SSR。同時改進圖片處理，並推出基於 WebAssembly 的 SWC 插件系統。

[我的超棒 HTML 郵件工作流程](https://www.joshwcomeau.com/react/wonderful-emails-with-mjml-and-mdx/?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Josh Comeau 解釋他如何用 MDX 撰寫郵件，再透過 MJML、React 和 Next.js 生成最終的 HTML 輸出，確保在所有郵件客戶端都能正常顯示。這些郵件透過 ConvertKit 發送，同時也會在網頁上公開。

[Remix 的數據流動](https://remix.run/blog/remix-data-flow?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

React 的初始模型可總結為 `ui = f(state)`。問題在於此模型未充分考量網絡同步（即 API 呼叫）。Jim 解釋 Remix 如何擴展 React 模型納入網絡層，從而減少對本地狀態管理的需求。

[你可能不需要使用 Effect](https://beta.reactjs.org/learn/you-might-not-need-an-effect?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

由 Dan Abramov 撰寫的新文件頁面，已合併至測試版網站。Effect 用於與外部系統同步，文中以具體案例和挑戰展示多種反模式。即使是資深開發者也應閱讀，部分內容可能令人驚訝：例如 [在渲染期間使用 setState](https://twitter.com/sebastienlorber/status/1541793224561463297?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter) 😱。我特別喜歡 `useSyncExternalStore` 的範例。

[React 18 推薦的數據加載方式是什麼？](https://www.reddit.com/r/reactjs/comments/vi6q6f/what_is_the_recommended_way_to_load_data_for/?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Dan Abramov 在 Reddit 解釋為何透過 useEffect 獲取數據並非理想方案。但若應用程式無 UX 問題則無需立即重寫：這些問題在 React 18 並非新出現，只是文件更完善。另可參閱關於 React-Native 數據獲取的評論。
