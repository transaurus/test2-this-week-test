## 📱 React-Native {#react-native}

本專欄現由 [Benedikt](https://twitter.com/bndkt) 共同撰寫。歡迎透過電子郵件或 [Twitter](https://twitter.com/bndkt) 向我們提供意見！

過去一週，我們暫時避開了重大的 React Native 頭條新聞（或是我漏掉了什麼？歡迎 [@我](https://twitter.com/bndkt)）。在經歷了 Expo 50、visionOS、React Native 現狀報告、react-strict-dom 等風暴後，這或許是個受歡迎的轉變。然而，這並不意味著 React Native 世界停滯不前。許多小公告揭示了 React Native 的近期未來：[RN 0.74](https://github.com/facebook/react-native/blob/ee2bc59dfe4579cad3b7f0b5de667a89fcac8710/CHANGELOG.md#0740-rc0) 的首批候選版本已發布，包含大量改進以及一些棄用和破壞性變更，但初步看來沒有需要擔心重大升級成本的部分。最值得注意的是，bridgeless 模式在 0.74 版成為新預設（僅當啟用新架構時）。Bridgeless 模式是新架構的第三大支柱，緊接已推出的 TurboModules 和新的渲染器（Fabric）。談到新架構和預設值，一個[PR](https://github.com/facebook/react-native/pull/43135) 已合併，將從 RN 0.75 開始使新架構成為預設（仍可選擇退出）。此外，定義 [React Native 框架](https://github.com/react-native-community/discussions-and-proposals/blob/main/proposals/0759-react-native-frameworks.md) 的 RFC 也已合併，明確劃分 RN 與基於 RN 的框架的職責。
