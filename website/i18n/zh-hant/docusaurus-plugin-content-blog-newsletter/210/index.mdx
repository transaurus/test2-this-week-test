## ⚛️ React {#react}

[![React 19 RC1](/emails/issues/210/react19.jpg)](https://github.com/facebook/react/issues/29898#issuecomment-2477449973)

[**React 19 RC1 - 兄弟元件預熱機制**](https://github.com/facebook/react/issues/29898#issuecomment-2477449973)

我們先前報導過 React 19 初始版本因 Suspense 變更導致中止發布的問題——該變更會在使用 Suspense 但未在事件處理器中啟動資料獲取的應用中引發瀑布流問題。  
React 團隊持續努力重新引入 Suspense 兄弟元件預渲染機制，同時確保不會對早期啟動資料獲取的應用造成效能衰退。他們的解決方案現已就緒，稱為[預熱機制 (pre-warming)](https://react.dev/blog/2024/04/25/react-19-upgrade-guide#improvements-to-suspense)。最先提出此問題的 React Query 維護者 Dominik [已確認對此修復方案感到滿意](https://bsky.app/profile/tkdodo.eu/post/3layw6wk5bs2b)。  
如同往常，新功能已提前在 Next.JS（採用 React Canary）中提供，包含伺服器元件與 Actions、新 `use` 鉤子、表單處理改進（`useFormStatus` 與 `useOptimistic`），以及一個微小但期待已久的變更：不再需要 `forwardRef`！
