## ⚛️ React 動態 {#react}

[![全新 react.dev 登場](/emails/issues/139/reactdev.jpg)](https://react.dev/blog/2023/03/16/introducing-react-dev)

**[隆重推出 react.dev](https://react.dev/blog/2023/03/16/introducing-react-dev)**

新版 React 官網正式上線！伴隨這次發布的專文詳述了這個龐大專案（涵蓋技術與內容），歷經多年努力與眾多協作者共同完成。

個人特別欣賞以下亮點：

- 聚焦於 Hooks 與函數式元件
- 指南包含豐富圖表、插圖與互動式挑戰
- 強調 React 作為跨平台解決方案（非僅限網頁）
- API 頁面明確區分參考/用法、當前/棄用、客戶端/伺服端
- 整體體驗的高度一致性

本週部分變動也引發討論，特別是官方僅推薦少量框架作為新 React 專案起點（Next.js、Remix、Gatsby、Expo），而未推薦其他常見方案（如 Vite、Create-React-App、客戶端 SPA 等）。

- 👀 [React PR - 暫停提交而不阻擋渲染](https://github.com/facebook/react/pull/26398)：此功能將首先在內部使用，特別針對資源（CSS、字體、圖像）。當使用 startTransition 導航時，React 能夠等待下頁圖像載入後再提交過渡（[詳情](https://twitter.com/acdlite/status/1637102880850059264)）。
- 👀 [React PR - 移除伺服器端的 layout effect 警告](https://github.com/facebook/react/pull/26395)：SSR 中 useLayoutEffect 的警告將被移除。Dan Abramov 評論了 [useIsomorphicLayoutEffect](https://twitter.com/dan_abramov/status/1636176793592905729)。
- 🧵 [Tim Neutkens 談改進 `next export`](https://twitter.com/timneutkens/status/1636693508223270912)：相容 App Router、伺服器元件...
- 📜 [為 Remix 應用進行未來驗證](https://remix.run/blog/future-flags)：Remix 闡述其重大版本中部署破壞性變更的策略。他們計劃引入「未來標誌」，允許預先選擇採用計劃中的破壞性變更。若在 v1 中啟用所有標誌，升級到 v2 將更輕鬆。
- 📜 [在 Next.js 13 中使用 React 伺服器元件實現國際化](https://www.smashingmagazine.com/2023/03/internationalization-nextjs-13-react-server-components/)：伺服器元件將對 React 應用的翻譯方式產生重大影響，尤其在效能、程式碼拆分和套件大小方面。本文對此主題有良好介紹。我在[此討論串](https://twitter.com/sebastienlorber/status/1638528549056311297)提供了更多細節。
- 📜 [你可能不需要的 Hooks](https://reacttraining.com/blog/hooks-you-probably-dont-need)：介紹 React 中較少人知的 hooks，並按需求可能性分類（哈囉 `useInsertionEffect` 😅）。我喜歡最後一節展示常見反模式及 `useLayoutEffect` 如何預防它。我認為 [useSyncExternalStore 非常適合](https://twitter.com/sebastienlorber/status/1615329010761842689)讀取瀏覽器 API（如 `localStorage`）。
- 📜 [Front-Commerce 正在轉向 Remix](https://developers.front-commerce.com/changelog/front-commerce-is-getting-a-remix)：繼 Shopify 後，Front-Commerce 決定基於 Remix 重構。這帶來許多優勢，例如能使用漸進增強、MPA 模式，並降低與 React 的耦合度。
- 📜 [Astro：新 JavaScript 框架有多好？](https://cult.honeypot.io/reads/astro-the-new-javascript-framework/)
- 📜 [透過遷移至 Astro 減少 Open UI 網站 97% 的 JavaScript](https://blog.andri.co/023-migrating-open-ui-site-from-gatsby-to-astro/)
