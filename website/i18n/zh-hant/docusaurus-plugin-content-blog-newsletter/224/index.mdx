## ⚛️ React {#react}

[![Next.js 15.2](/emails/issues/224/next.jpg)](https://nextjs.org/blog/next-15-2)

[**Next.js 15.2**](https://nextjs.org/blog/next-15-2)

Vercel 發布 Next.js 新版次，大幅改善開發體驗，重點更新包括：

- 重新設計錯誤 UI 與強化堆疊追蹤：採用 React 全新 [`captureOwnerStack()`](https://react.dev/reference/react/captureOwnerStack) API 的除錯體驗
- 串流中繼資料：非同步中繼資料不再阻擋頁面渲染或客戶端頁面轉換
- Turbopack 效能提升：編譯加速與記憶體用量降低
- React 視圖轉換（實驗性）：支援 React 新視圖轉換 API
- Node.js 中介軟體（實驗性）：支援在中介軟體中使用 Node.js 運行環境

[**TanStack Form 1.0**](https://tanstack.com/blog/announcing-tanstack-form-v1)

歷經多年開發，TanStack 全新函式庫正式發布穩定版。公告文章重點介紹其特色功能：

- 框架無關與無頭設計：支援 5 種框架，文章範例採用 React
- 極致型別安全：創新 `<form.Field>` API 設計
- 結構描述驗證：支援 Zod、Valibot、ArkType 及標準結構描述
- 非同步驗證、內建防抖動、AbortSignal 中斷機制
- 同構多平台：可共享客戶端/伺服端驗證邏輯

## 📱 React-Native {#react-native}

[![Lynx：解鎖原生，實現更多](/emails/issues/224/lynx.jpg)](https://lynxjs.org/blog/lynx-unlock-native-for-more)

[**Lynx：解鎖原生，實現更多**](https://lynxjs.org/blog/lynx-unlock-native-for-more)

位元組跳動剛剛宣布了 Lynx，他們的原生跨平台技術。這項技術被廣泛應用於他們的生產應用程式中，包括 TikTok！透過 React 綁定，它成為了 React Native 的強勁競爭者，並且預期將支援其他框架。其架構選擇相當有趣，採用了雙執行緒模型和不同的執行環境。它支援一些連 React Native 尚未具備的 CSS 和 Web API，並使用了方便的內建 JSX 內建元素，例如 `<view>`。

發布時的功能組合令人印象深刻，但也有一些你可能不喜歡的驚人之處，例如使用 React 17 / Preact，或在網頁上使用自訂元素。時間將證明這項技術是否能兌現承諾，並真正與現有的主要競爭者匹敵。無論如何，競爭是健康的。
