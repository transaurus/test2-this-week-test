## 📱 React-Native {#react-native}

本節現由 [Benedikt](https://twitter.com/bndkt) 共同撰寫。歡迎透過電子郵件或在 [Twitter](https://twitter.com/bndkt) 上向我們發送您的評論！

[![2023 年 React Native 現狀 - 調查結果](/emails/issues/172/stateofrn.jpg)](https://results.stateofreactnative.com/)

**[2023 年 React Native 現狀 - 調查結果](https://results.stateofreactnative.com/)**

我是所有「現狀」調查的忠實粉絲，因為它們讓我們獨家洞察生態系統的動態。在 X（前身為 Twitter）和其他地方追隨熱潮是一回事，但看到開發者實際使用的統計數據又是另一回事。因此，我非常高興看到去年的 React Native 現狀調查結果發布！由於這是該調查進行的第二年，我們還可以透過與前一年的結果進行比較來識別一些趨勢。有關所有詳細資訊，請[自行查看調查結果](https://results.stateofreactnative.com/)！我將重點介紹一些我認為有趣的事情，其中一些真的讓我感到驚訝：

- 狀態管理持續處於非常有趣的狀態（雙關語），其中 Redux 在保留率方面以僅 44%（去年為 57%）位居末位，但在使用率方面仍以 78%（去年為 85%）緊隨 React 原生功能之後位居第二。
- 資料獲取類別的結果與在社群媒體上觀察到的關注程度一致，其中 TanStack Query 和 tRPC 在興趣和保留率方面位居前列，而 Meta 自家的 Relay 在 Meta 之外基本無關緊要，使用率僅為 5%。
- 導航的未來是 Expo Router，其使用率從去年的 12% 飆升至如今的 40%。
- 根據社群媒體活動判斷，樣式似乎是討論最多的類別，選項豐富。在使用方面，經典選項（StyleSheet API、內聯樣式、styled components）仍占主導地位，而 NativeWind 和 Tamagui 正在迎頭趕上。Tamagui 在過去一年中成功引起了最多的興趣（同時作為 UI 元件庫擁有最高的保留率和興趣），緊隨其後的是一個新的競爭者：react-native-unistyles。
- 特別點名 Expo：在「React Native 工具」部分中，前 10 大工具中有 6 個以及部署類別中的前 2 個選項均由 Expo 開發。
- 超過 85% 的受訪者認為 React Native 正朝著正確的方向發展，且認為目前建構 RN 應用程式過於複雜的開發者比例從約 30% 降至 25%——這仍然太高，但我們似乎正朝著正確的方向前進！

總體而言，這些結果非常有趣，值得[詳細查看](https://results.stateofreactnative.com/)！非常感謝 Bartłomiej、Kacper 和 Software Mansion 讓這一切成為現實！
