## ⚛️ React {#react}

[![StyleX](/emails/issues/165/stylex.jpg)](https://github.com/facebook/stylex)

[**StyleX**](https://github.com/facebook/stylex)

Meta 的 CSS-in-JS 解決方案經過漫長等待終於開源，已在 Facebook、Instagram 和 WhatsApp 等主要應用中實際使用。它並非嚴格與 React 綁定，原子化 CSS 將在編譯時提取，並透過輕量級運行時拼接類別。TypeScript 支援經過精心設計，API 極簡主義，主要依賴兩個函數：`create` 和 `props`。我們將觀察這個新方案在已有 Tailwind、Panda 等競爭對手的生態系中如何定位。

另請參閱：

- 🎥 [Jack Herrington - StyleX：Meta 的大規模 CSS 解決方案](https://www.youtube.com/watch?v=6ZDiGtg1jN4)
- 🎥 [Theo T3 - Facebook 嘗試 Tailwind 後，改為打造此方案](https://www.youtube.com/watch?v=PHV94q7BivI)

[![CSS Hooks](/emails/issues/165/csshooks.jpg)](https://css-hooks.com/)

具備 React 綁定的 CSS 函式庫，透過行內樣式實現真正的回歸基礎。作者在[從 CSS 混亂到 CSS Hooks](https://nsaunders.dev/posts/css-madness-to-hooks) 中解釋如何運用變數 CSS 技巧支援媒體查詢、偽類選擇器等缺失功能。

個人而言，我一直偏好行內樣式的開發體驗，但實務上功能過於受限且效能表現不穩定。此創新方案能否經得起實踐考驗尚待觀察。Mark Dalgleish (🐦 [推文](https://twitter.com/markdalgleish/status/1729399475494608923)) 對此感到興奮。值得注意的是，若行內樣式支援更完善，Adam Wathan 可能就不會創造 Tailwind (🐦 [推文](https://twitter.com/adamwathan/status/1732209400641724785))。
