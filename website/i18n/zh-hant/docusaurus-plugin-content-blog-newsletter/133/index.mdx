## ⚛️ React {#react}

[![Astro 渲染 v2](/emails/issues/133/astro-rendering.jpg)](https://astro.build/blog/hybrid-rendering/)

**[Astro - 透過混合渲染開啟全新可能性](https://astro.build/blog/hybrid-rendering/)**

Astro 1 需全站統一選擇：100% 靜態 (SSG) 或 100% 伺服器渲染 (SSR)。Astro 2.0 現可逐頁設定。本文詳述 Astro 建置的內部運作，以及為支援此功能所做的改進。他們透過靜態分析區分靜態路由與 SSR 路由。

[![React 為何不會消亡](/emails/issues/133/reactisntdying.jpg)](https://tkdodo.eu/blog/why-react-isnt-dying)

**[React 為何不會消亡](https://tkdodo.eu/blog/why-react-isnt-dying)**

Dominik 闡釋 React 不會消亡的原因。React 推廣了 JSX、組件概念與聲明式 UI。替代方案雖借鑑優點試圖解決問題，但考量其龐大生態系及持續壯大的社群，挑戰難度極高。這需要突破性創新。框架無關的函式庫（如 TanStack）扮演重要角色，促進跨生態系知識轉移。

[![可恢復的 React：如何在 Qwik 中使用 React](/emails/issues/133/qwik.jpg)](https://www.builder.io/blog/resumable-react-how-to-use-react-inside-qwik)

**[可恢復的 React：如何在 Qwik 中使用 React](https://www.builder.io/blog/resumable-react-how-to-use-react-inside-qwik)**

本文重點介紹了 Qwik 與 React 之間的整合。Qwik 是一個處於測試階段的創新框架，基於可恢復性（resumability）而非水合（hydration）。它提供了一個方法 `qwikify$()`，允許將現有的 React 元件改造成可在 Qwik 應用程式中使用，並提供多種水合模式。為了盡可能減少套件大小，可以避免對 React 元件進行水合，並直接使用原生的 DOM 事件。
