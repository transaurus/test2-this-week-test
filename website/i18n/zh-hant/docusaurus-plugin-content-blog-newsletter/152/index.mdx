## ⚛️ React {#react}

[![React 18 如何提升應用程式效能](/emails/issues/152/react18perf.jpg)](https://vercel.com/blog/how-react-18-improves-application-performance)

**[React 18 如何提升應用程式效能](https://vercel.com/blog/how-react-18-improves-application-performance)**

關於 React 18 及其對效能影響的全面且圖文並茂的文章。解釋了並行 React 的工作原理及其對總阻塞時間（Total Blocking Time）和交互到下一次繪製（Interaction to Next Paint）等指標的影響。同時也介紹了其他新功能，例如過渡 API（Transition API）、React 伺服器元件（React Server Components）、Suspense 以及新的 React 快取（cache）。

---

[![Kuma UI](/emails/issues/152/kuma.jpg)](https://www.kuma-ui.com/)

**[Kuma UI](https://www.kuma-ui.com/)**

新的 CSS-in-JS 函式庫，靈感來自 Chakra UI 提供的開發者體驗（DX），並與 React 伺服器元件相容。該函式庫基於一種獨特的混合方法，結合了零執行時 CSS-in-JS 和傳統執行時解決方案的優點。我們必須研究這一切，並看看這個函式庫如何與來自 Chakra 團隊的 [Panda CSS](https://panda-css.com/) 相抗衡。
