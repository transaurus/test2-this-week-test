## ⚛️ React {#react}

**[React 還有未來嗎？](https://www.infoxicator.com/is-react-going-anywhere)**

Ruben 在遇到一位放棄 React 轉用 Vanilla JS 的電商 CTO 後提出反思：應用效能低下是否真該歸咎 React？改用替代方案是否合理？React 在未來 5-10 年能否保持領導地位？

**[將 React 互動效能提升 4 倍](https://www.causal.app/blog/react-perf)**

Causal 的雲端試算表應用遭遇效能問題。他們透過 Chrome 分析工具詳細解析問題根源與解決方案：部分問題來自 AG Grid（已提交修補），其餘源於自身代碼的記憶化處理。嘗試使用 Web Workers 時，序列化成本過高導致不可行。

- 📜 [逐步增強 Remix 中 useFetcher 鉤子的功能](https://sergiodxa.com/articles/progressively-enhance-the-usefetcher-hook-in-remix)：在 Remix 的 useFetcher API 上支援有/無 JavaScript 使用者的小技巧：對無 JavaScript 使用者強制重定向。同樣地，也可以處理[檔案上傳的漸進增強](https://twitter.com/ebey_jacob/status/1596223682246234113)。這些功能對多數應用可能非必需，但了解其可行性很有價值。
- 📜 [Remix Deferred 概覽](https://gist.github.com/jacob-ebey/9bde9546c1aafaa6bc8c242054b1be26)：解析 Remix 中 [defer](https://github.com/remix-run/remix/issues/4486) 的運作原理，可將 promise 從後端「傳送」至前端。適合想深入理解 Remix 內部機制者。
- 📜 [如何在 React 中更新兄弟組件狀態？](https://alexsidorenko.com/blog/react-siblings/)：簡潔闡釋「狀態提升」概念的短文。
- 📜 [React 開發者必備的 TypeScript 速成課程](https://profy.dev/article/react-typescript)：TypeScript 與 React 的入門指南。
- 📜 [前端 JavaScript 真實場景中的資料結構 - 附 React 程式碼範例](https://profy.dev/article/javascript-data-structures)：在 React 環境中熟悉資料結構。
- 📜 [ArConnect 將 Chrome 擴充功能遷移至 Plasmo 框架](https://blog.plasmo.com/p/migrating-chrome-extension-to-plasmo)：使用 [Plasmo](https://github.com/PlasmoHQ/plasmo) 的實戰回饋，此開源 React 框架用於建立網頁擴充功能。
- 📜 [給全端開發者的 Storybook](https://storybook.js.org/blog/storybook-for-full-stack-developers/)：完整介紹 Storybook 生態系的應用場景。
- 📜 [Solid Final Form - 概念驗證](https://erikras.com/blog/solid-final-form-poc)：Erik 說明其函式庫 [Final Form](https://final-form.org/react)（數年前流行）與 React 解耦，並展示在 SolidJS 環境的應用可能。
- 📜 [Ant Design 5.0 - 組件級 CSS-in-JS](https://ant.design/docs/blog/css-in-js)：React 設計系統發布重大更新，說明其優化 CSS-in-JS 運行時的方法。
- 🎥 [論 React 與單頁應用的優點與局限](https://www.youtube.com/watch?v=ChfjZV-aA_Y)：Alex Russell 與 Theo Browne 的 2 小時辯論。Alex 以批判 React 和 SPA 聞名，Theo 則是 React 生態系的強力捍衛者。
- 🎥 [React 開發者必須了解的 @container！](https://www.youtube.com/watch?v=ypN-Uwshc5M)：以 React 環境結合原生 CSS + Tailwind 介紹容器查詢（container queries）新特性。Firefox 尚不支援，但有 polyfill 可用。
- 🎥 [RTK Query 基礎：查詢端點、數據流與 TypeScript](https://egghead.io/courses/rtk-query-basics-query-endpoints-data-flow-and-typescript-57ea3c43)：RTK Query 核心開發者 Lenz 在 Egghead 的免費新課程。
- 🧑‍💻 [Remix PR - 新增「跳至定義」支援](https://github.com/remix-run/remix/pull/4689)：tRPC 貢獻者將其 TypeScript 經驗移植至 Remix 以提升開發體驗。
- 📦 [react-three-editor](https://github.com/pmndrs/react-three-editor)：開發中的 React-Three-Fiber 場景編輯器。
- 📦 [Gatsby 5.1](https://www.gatsbyjs.com/docs/reference/release-notes/v5.1/)
