[![Remix 覺醒](/emails/issues/236/remix.jpg)](https://remix.run/blog/wake-up-remix)

**[Remix 覺醒 - v3 版本規劃](https://remix.run/blog/wake-up-remix)**

本週您可能聽聞關於 [Remix 將棄用 React](https://x.com/AdamRackis/status/1925646909345058818) 的傳言。Ryan 和 Michael 剛發布官方文章闡述 Remix 的未來。

Remix v3 的目標是「重新構想 Web 框架的可能性」。文章闡述了新框架將遵循的核心原則（為大型語言模型優化、基於 Web API 構建、組合式架構等）。

**[Clerk 帳務系統入門指南](https://go.clerk.com/KfnV7Rj)**

學習如何使用 Clerk 打造完整帳務體驗——無需自訂 UI、Webhooks 或 Stripe 會話管理。

- 使用 `<PricingTable />` 元件建立定價頁面
- 與 Stripe 連接 — 無需手動 API 呼叫，無需手動同步
- 在 Clerk 儀表板內統一管理方案和權益
- 使用 `has()` 輔助函式根據客戶的方案、功能和權限控制存取
- 在單一位置管理使用者和訂閱資料

Enable Billing, drop in `<PricingTable/>` on your `/pricing` page and [start monetizing your application](https://go.clerk.com/LphCouG).
