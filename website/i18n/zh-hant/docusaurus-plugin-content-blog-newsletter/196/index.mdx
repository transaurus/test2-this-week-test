## 📱 React-Native {#react-native}

[![React Native WebGPU](/emails/issues/196/rn-webgpu.jpg)](https://www.youtube.com/watch?v=i-DGY98cTjk)

**🎥 [React Native WebGPU](https://www.youtube.com/watch?v=i-DGY98cTjk)**

React Native WebGPU 已正式推出，提供統一 API 來運用 GPU 實現進階圖形與計算功能！透過重用 Web API 標準，我們能輕鬆整合現有 WebGPU 程式碼範例與函式庫（如 ThreeJS）至 React Native。
除了圖形處理，React Native WebGPU 還能用於矩陣運算等任務，使其在裝置端機器學習演算法中極具潛力。該專案採用 Nitro Modules 建構，實現 TypeScript 與 C++ 的無縫互通。目前處於技術預覽階段，預示著令人振奮的發展前景！

---

**React Native 模組的未來**

React Native 開發者的激動時刻！Marc Rousavy 正透過 NitroModules 革新原生程式碼開發方式 📦 [在速度測試中大幅超越現有方案](https://github.com/mrousavy/NitroBenchmarks)。他在 VisionCamera 中發現了 JSI 的某些限制。該專案在 C++ 端已趨穩定，並應用於多個專案如 🐦 [react-native-unistyles](https://x.com/jpudysz/status/1822541758137303233) 和 react-native-webgpu，Swift 與 Kotlin 綁定仍在開發中。

👀 [與此同時，一項推動 Swift 用於 TurboModules 的新提案已提出](https://github.com/okwasniewski/discussions-and-proposals/blob/22edd2c7853aaf2ee654b57f4521e43d668103fb/proposals/0000-swift-turbo-modules.md)。這些變更將透過棄用 Objective-C 和 C++ 來簡化原生模組開發。隨著這些更新，建置原生模組將大幅提升效率！
