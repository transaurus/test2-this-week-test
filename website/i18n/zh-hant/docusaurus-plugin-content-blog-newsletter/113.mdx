## React 核心

[**原始碼導讀：React**](https://alexkondov.com/readint-source-code-react/?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Alex 解讀 React 原始碼並針對多個部分進行註解，以 React 應用程式的掛載過程為切入點。內容包含原型使用、JSX 運作機制，以及（通用型）協調器與 DOM 渲染器互動等技術細節。

[**我們能否都承認 React Hooks 是個壞主意？**](https://medium.com/codex/can-we-all-just-admit-react-hooks-were-a-bad-idea-c48120c5188d?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Amy 認為 React hooks 違反了 SOLID 架構原則。我並不完全同意，但承認網路上大多數範例都是簡化版，未必能展現良好架構。

[**在 React 中應用 SOLID 原則**](https://konstantinlebedev.com/solid-in-react/?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

幾乎是對上述文章的直接回應，儘管此處 SOLID 的應用涵蓋更廣（元件 + hooks）。注意：別猶豫使用 React context 進行依賴注入。

## React-Native

[**Hermes 成為預設引擎**](https://reactnative.dev/blog/2022/07/08/hermes-as-the-default?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Hermes JS 引擎是 JSC/V8 的替代方案，透過位元組碼預編譯加速應用啟動。React-Native 0.70（現為[發布候選版](https://github.com/facebook/react-native/releases/tag/v0.70.0-rc.0?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)）已預設啟用 Hermes。Michael 分享開源應用 Mattermost 的多元效能測試。Hermes 分發模式已調整以避免 ABI 相容問題，iOS 補齊缺失的 Intl API，並將支援 BigInt 與 WeakRef。

[**我在 Airbnb 的 React Native 時代工作所學到的關於 web3 的經驗**](https://medium.com/@devinabbott/what-working-at-airbnb-during-its-react-native-era-taught-me-about-web3-6d34d855d892?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Devin 回顧了 React-Native 在 Airbnb 的歷史，以及決定停止使用它的細微考量。他將此與 web3 做了有趣的類比。從長遠來看，技術的潛力勝過早期採用者的痛苦。
