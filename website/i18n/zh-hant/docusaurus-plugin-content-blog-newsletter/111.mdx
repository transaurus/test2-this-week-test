## React 生態

[**Reassure**](https://github.com/callstack/reassure?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Reassure 是新的 React 效能測試函式庫。由 Callstack 開源，能在偵測到效能衰退時於 Pull Request 自動提交報告，例如元件渲染變慢或渲染次數異常增加。目前僅支援 React-Native，但網頁版支援即將推出。

[**如何實現多人游標動畫**](https://liveblocks.io/blog/how-to-animate-multiplayer-cursors?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

精彩的互動式文章，探討跨瀏覽器即時多人游標動畫實作。問題比表面更複雜，每種解決方案都搭配 React 實例展示不同權衡：CSS 過渡、彈簧動畫、曲線擬合...

[**Next.js Middleware 實戰指南**](https://www.ctnicholas.dev/articles/how-to-use-nextjs-middleware?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

同作者的互動式文章，展示多種中介層應用場景。因應 Next.js 12.2 正式版破壞性變更，本文已同步更新。

[**Meteor 十年回顧**](https://meteor10.sachagreif.com/?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Sacha Grief（《Discover Meteor》合著者，同時也是 State of JS、CSS 調查的發起人...）回顧了他與這個框架長達十年的個人歷程。Meteor 雖已不再流行，但仍被使用中，並對我們的前端生態系統產生了深遠影響。

[**React 元素、子元件、父元件與重新渲染之謎**](https://www.developerway.com/posts/react-elements-children-parents?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Nadia 揭示了 React 中某些令人驚訝甚至違反直覺的行為，並解釋了在哪些邊緣情況下父元件的重新渲染會觸發子元件的重新渲染。

[**Storybook 社群精選 #2**](https://storybook.js.org/blog/community-showcase-2/?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Storybook 社群最新動態摘要：元件百科全書、Figma 外掛、Story Explorer，以及各種擴充功能：變體、CSS 變數、Recoil...

[**React 狀態管理的新浪潮**](https://frontendmastery.com/posts/the-new-wave-of-react-state-management/?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

探討狀態管理庫需解決的各種問題，並進行對比分析。包含 Redux 前後的發展回顧。內容完整但篇幅較長：閱讀門檻較高。

## React-Native

[**FlashList - 高效能 React Native 列表元件**](https://shopify.github.io/flash-list/?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

Shopify 近期發布 FlashList，作為 FlatList 的替代方案，解決滾動效能問題並避免顯示空白單元格。目標是在低階 Android 設備上維持 60fps 流暢度，同時保持實作簡潔。此函式庫會複用已渲染單元格：您需調整 React `key` 的使用方式以充分發揮其效能。

[**React-Native-Compat-Table**](https://retyui.github.io/react-native-compat-table/?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)

以全面方式列出每個 React-Native 運行環境（JSC、V8、Android...）的 API 支援情況。作者同時解釋了[為什麼 React Native 推薦 TS 設定庫使用 "es2017"](https://dev.to/retyui/why-react-native-typescript-config-recommends-to-use-a-five-years-old-standard-lib-es2017-3bjk?utm_campaign=This%20Week%20In%20React&utm_medium=email&utm_source=Revue%20newsletter)
