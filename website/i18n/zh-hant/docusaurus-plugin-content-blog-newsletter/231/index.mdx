## 📣 來自 React 團隊 {#react-core-team}

[![React Labs: View Transitions, Activity, and more](/emails/issues/231/labs.jpg)](https://react.dev/blog/2025/04/23/react-labs-view-transitions-activity-and-more)

**[React Labs：檢視轉換、活動等新功能](https://react.dev/blog/2025/04/23/react-labs-view-transitions-activity-and-more)**

關於 React 團隊只關注伺服端的傳言，實屬過度誇大。

在今天的 [React Labs 文章](https://react.dev/blog/2025/04/23/react-labs-view-transitions-activity-and-more)中，我們宣布 [View Transitions](https://react.dev/reference/react/ViewTransition) 和 [Activity](https://react.dev/reference/react/Activity) 功能已可在實驗頻道試用，並附有對應文件。這些新功能讓客戶端運行的 React 能實現 UI 轉場動畫，並在使用者進行客戶端導航時動態顯示/隱藏 UI 區塊。它們雖能與伺服端功能協作，但主要解決客戶端問題。

我們也[更新了](https://react.dev/blog/2025/04/23/react-labs-view-transitions-activity-and-more#features-in-development)影響客戶端渲染應用的重點開發領域：包括強化效能分析工具、自動編譯 effect 依賴項、潛在的 IDE 擴充功能、片段引用(fragment refs)，甚至並行儲存機制(concurrent store)。這些長期存在的痛點，正因 React Compiler 和 `use` API 等新功能而得以解決。

這些功能尚處早期階段，正式發布前可能大幅變動。您可持續關注開發進度，本通訊是掌握最新動態的理想管道。無須急於學習所有內容——請先讓我們專心開發。

當我們對功能發布更具信心時，將如同今日的 View Transitions 和 Activity 一樣，在部落格正式公告並發布文件。

試用新的實驗性功能，並在[部落格文章](https://react.dev/blog/2025/04/23/react-labs-view-transitions-activity-and-more)中查看我們正在開發的領域。

\-[Ricky Hanlon](https://bsky.app/profile/ricky.fm)

## 📱 React-Native {#react-native}

[![Legend List 1.0](/emails/issues/231/legend-list.png)](https://bsky.app/profile/jayz.us/post/3lni5ofbuvs2s)

**[Legend List 1.0](https://bsky.app/profile/jayz.us/post/3lni5ofbuvs2s)**

實現長列表的高效能渲染是個艱難挑戰，而 React Native 舊架構因非同步佈局計算使問題更加棘手。我們記得 Margelo 曾放棄其 `WishList` 專案。
Legend List 現已發布 1.0 版，承諾比 `FlashList` 更快速，同時能處理「複雜案例」：雙向無限列表、可變項目尺寸... 且兼容舊架構（儘管效能較低且可能出現閃爍）。

正如 Seb 上週提及，[`FlashList` v2](https://github.com/Shopify/flash-list/tree/new-rlv-prototype) 也正在開發中，將更易使用（例如無需指定 estimatedItemSize）。

所幸這些函式庫皆複製 `FlatList` 的 API，因此能輕鬆切換使用 🙂。
