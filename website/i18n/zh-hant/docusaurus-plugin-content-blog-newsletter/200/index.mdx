## ⚛️ React {#react}

[![Josh Comeau - CSS `:has` 的無可否認實用性](/emails/issues/200/css-has.jpg)](https://www.joshwcomeau.com/css/has/)

**給 React 開發者的 CSS `:has`**

本週有兩篇關於 CSS `:has` 選擇器的精彩文章，探討 React 開發者如何運用它來簡化/刪除 JS 代碼並提升效能。這項 CSS 新功能支援度良好（92%），但無法填充（polyfill），因此建議目前先用於細節優化與漸進增強。

按此順序閱讀：

- 📜 [Josh Comeau \- CSS `:has` 的無可否認實用性](https://www.joshwcomeau.com/css/has/)：精彩的互動式入門文章，包含對 React 開發者相關的評論與範例，例如使用 data 屬性而非 `useEffect` 來阻止模態框開啟時的全局頁面滾動。
- 📜 [Nadia Makarevich \- 用 CSS `:has` 選擇器取代 React 程式碼](https://www.developerway.com/posts/replacing-react-with-css)：提供更多基於 React 的實用範例，例如處理焦點狀態、根據輸入狀態高亮表格行或表單欄位群組。

能力越大，責任越大。請保持 React 元件的封裝性，並盡可能避免樣式洩漏至元件邊界外。

## 📱 React-Native {#react-native}

上週，弗羅茨瓦夫成為 React Universe 的舞台，重大公告令 React Native 社群振奮。其中最引人注目的是 React Native 0.76 的發布預告，預計十月底推出。此版本帶來重大進展，包括預設啟用新架構及全面翻新的 React Native 開發者工具。目前已可作為 [0.76 候選版本](https://github.com/facebook/react-native/releases/tag/v0.76.0-rc.0)\ 使用！

[![React Native 開發者工具](/emails/issues/200/react-native-devtools.jpg)](https://www.youtube.com/watch?v=b48Lax2-jOQ&t=4350s)

- 🎥 [Alex Hunt - 全新 React Native 調試工具 - 代號 "Fusebox"](https://www.youtube.com/watch?v=b48Lax2-jOQ&t=4350s)
  React Native 的調試工具長期處於分散狀態，但新的 "React Native DevTools" 透過基於 Chrome 開發者工具協議 (CDP) 的新後端實現統一體驗。此架構獲 IDE 和 Chrome DevTools 支援，承諾提供更可靠的斷點和穩定的重新載入功能。它同時支援新舊架構，成為 React Native 應用的首選調試方案。直接透過 Chrome 調試 Hermes 的方式將逐步淘汰，儘管性能和網絡檢查功能仍在開發中。

[![Hermes 性能圖表](/emails/issues/200/hermes-perf.jpg)](https://www.youtube.com/watch?v=pfeTPSai6vc&t=8702s)

- 🎥 [Tzvetan Mikov - Hermes: Better performance with runtime bytecode translation](https://www.youtube.com/watch?v=pfeTPSai6vc&t=8702s)  
  去年在 React Native Europe 推出的 Static Hermes，透過挑戰現有 JavaScript 引擎標準引發了廣泛關注。雖然仍在開發階段，但 Static Hermes 提供了一種創新方法：透過類型註解分析將 JavaScript 編譯為原生機器碼或類型化字節碼，具備達到 C++ 級效能的潛力。然而，此工作流程存在限制：空中更新（OTA）將不再運作、現有 JavaScript 建構工具（如 Babel）無法保留類型註解，且仍有大量未類型化程式碼在使用中。為解決這些挑戰，Hermes 團隊推出了運行時字節碼轉譯技術，可將原生編譯效能提升高達 12 倍。這些創新將在 Hermes v2 中推出，待 Meta 完成全面內部測試後即可使用。
