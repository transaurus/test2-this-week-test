## ⚛️ React {#react}

[![Netlify 收購 Gatsby Inc. 以加速模組化網頁架構的採用](/emails/issues/134/netlify-gatsby.jpg)](https://www.netlify.com/press/netlify-acquires-gatsby-inc-to-accelerate-adoption-of-composable-web-architectures/)

**[Netlify 收購 Gatsby Inc. 以加速模組化網頁架構的採用](https://www.netlify.com/press/netlify-acquires-gatsby-inc-to-accelerate-adoption-of-composable-web-architectures/)**

這是本週重大新聞。Gatsby Inc. 被 Netlify 收購。該框架現歸入 Netlify 旗下，與 Eleventy 和 SolidJS 並列，並將持續發展。雲端功能（包括 [Valhalla 內容中心](https://www.gatsbyjs.com/products/valhalla-content-hub/)）將整合進 Netlify 的服務。新口號是「模組化架構」。Gatsby 目前並非最熱門的 React 框架，但我認為此次收購具有戰略意義，尤其考量其內容中心與所有 CMS 整合功能。相較於 Netlify 的選配付費功能，Gatsby Cloud 更難被採用，對於未使用 Gatsby 框架的網站尤其如此。

另請參閱：

- 📜 [Gatsby 加入 Netlify](https://www.gatsbyjs.com/blog/gatsby-is-joining-netlify/)
- 📜 [Netlify 收購困境中的 Jamstack 競爭對手 Gatsby](https://thenewstack.io/netlify-acquires-gatsby-its-struggling-jamstack-competitor/)
- 🎥 [為何有人要收購 Gatsby？](https://www.youtube.com/watch?v=5HE6FcOpit8)

[![在 React 中實現全域狀態機器](/emails/issues/134/xstate.jpg)](https://stately.ai/blog/making-state-machines-global-in-react)

**[在 React 中實現全域狀態機器](https://stately.ai/blog/making-state-machines-global-in-react)**

介紹 XState 3.1 中新增的 `createActorContext` API。該 API 能簡化全域 XState 狀態機的創建，並與 React context 關聯。可使用選擇器系統，或透過 provider 提供自訂狀態機（這對於從父組件 `props` 自訂狀態機非常便利）。

## 📱 React-Native {#react-native}

[![適用於原生的 React DOM（減少 API 碎片化）](/emails/issues/134/rn-web.jpg)](https://github.com/necolas/discussions-and-proposals/blob/reduce-fragmentation/proposals/0000-reduce-fragmentation.md)

**[適用於原生的 React DOM（減少 API 碎片化）](https://github.com/necolas/discussions-and-proposals/blob/reduce-fragmentation/proposals/0000-reduce-fragmentation.md)**

由 Nicolas Gallagher（Meta / React-Native-Web）提出的[RFC](https://github.com/react-native-community/discussions-and-proposals/pull/496) 已更新。目標是盡可能讓 React-Native API 與 Web API 對齊，包括 DOM 元素如 `div`、`span`，甚至 `select`！"一次學習，隨處運行" 可能逐步轉變為 "一次編寫，隨處運行"。社群反應相當分歧。我們需要時間沉澱思考這是否是個好主意 😄，考慮到這個雄心勃勃專案的規模，顯然不會明天就實現！
