## ⚛️ React {#react}

[![Pigment CSS](/emails/issues/177/pigment.jpg)](https://github.com/mui/material-ui/tree/master/packages/pigment-css-react)

[**Pigment CSS**](https://github.com/mui/material-ui/tree/master/packages/pigment-css-react)

MUI 團隊正在開發 Pigment，這是他們自研的零運行時 CSS-in-JS 解決方案，將在 Material UI v6 中取代 Emotion。首個版本已發布至 npm，包含 Next.js `withPigment()` 插件。Olivier 在 🐦 [推文](https://twitter.com/olivtassinari/status/1765719897152127315) 中解釋了創建新方案的動機，及其與現有方案（Panda、StyleX、Tailwind）的比較。與其他方案不同，此方案將降低現有 Emotion 和 Styled-Components 用戶的遷移門檻。

[![Storybook 8](/emails/issues/177/storybook.jpg)](https://storybook.js.org/blog/storybook-8/)

[**Storybook 8**](https://storybook.js.org/blog/storybook-8/)

此重大版本帶來諸多改進，包括：

- 實驗性 React Server Components 支援
- 更快的 React 控件自動生成（透過 `react-docgen`）
- 「測試模式」下更快的靜態建構（CLI 參數 `--test`）
- 內建視覺測試（透過 Chromatic）
- 重新架構的 Vite 支援，兼容 Vitest、Vite 5
- 改進的行動/桌面 UI/UX

## 💸 贊助商 {#sponsor2}

[![Refine - 開源 Retool 替代方案](/emails/issues/177/refine.jpg)](https://github.com/refinedev/refine)

**[Refine - 開源 Retool 替代方案](https://github.com/refinedev/refine)**

在低代碼與傳統開發之間找到完美平衡點，專為 CRUD 密集型應用打造。

作為專為企業應用設計的開源 React 元框架，[Refine](https://github.com/refinedev/refine) 提供無頭(headless)解決方案，非常適合用於管理面板、儀表板和內部工具等任務。

儘管低代碼工具初期看似有潛力，但由於可擴展性問題、自定義選項有限以及供應商鎖定風險，它們往往難以處理複雜情境。

Refine 讓團隊能在保持靈活性的同時，快速開發可直接投入生產的應用程式。
