## ⚛️ React {#react}

[![TanStack Router](/emails/issues/123/tanstack-router.png)](https://tanstack.com/router/v1)

**[TanStack Router](https://tanstack.com/router/v1)**

Tanner揭曉新專案：100%類型安全、框架無關的路由器。具備令人印象深刻的功能，包括：

- URL狀態管理
- 內建快取
- 搜尋參數結構/驗證
- 自動預取
- 載入器/動作
- 非同步元件+代碼分割

初步看來，靈感部分源自Remix或React-Router，但API設計差異顯著且不基於JSX，這可能有助實現框架無關與類型安全。目前僅有React適配器，文檔尚不完整，但前景相當看好！

🔗 **實用連結**：

- 🐦 [Tanner Linsley](https://twitter.com/tannerlinsley/status/1590101866385489929)在Twitter發布專案並附精彩宣傳片！
- 🐦 [Ryan Florence](https://twitter.com/ryanflorence/status/1590194770936463362)可參考其工作成果來提升Remix類型安全性。

[![Gatsby 5：迄今最快的 Gatsby](/emails/issues/123/gatsby5.png)](https://www.gatsbyjs.com/blog/gatsby-5/)

**[Gatsby 5：迄今最快的 Gatsby](https://www.gatsbyjs.com/blog/gatsby-5/)**

Gatsby 5.0 已正式發佈。它包含 Gatsby Slice API，可在修改佈局元素（如頁首、頁尾等）時提升建置速度；同時支援測試版的 Server Components，可實現部分水合作用（partial hydration）。此外還新增了 Head/Script 元件、GraphiQL v2 等功能。

Gatsby 還推出了 [Valhalla](https://www.gatsbyjs.com/products/valhalla-content-hub/)，這是其雲端平台上的新服務，旨在利用 Gatsby 來源插件建立內容中心與解耦的統一 GraphQL API。[Valhalla 也可與 Next.js 搭配使用](<https://github.com/gatsby-inc/valhalla-examples/blob/main/frameworks/nextjs/app/(pets)/page.tsx>)。

靜態網站生成器目前雖不熱門，但仍適用於許多網站。我認為至少有一個主要參與者持續在該領域創新是件好事。Slice API 看起來是個不錯的構想。我期望 Valhalla 能本地運行，但目前看來它僅是雲端服務。

## 📱 React-Native {#react-native}

[![Expo SDK 47](/emails/issues/123/expo47.png)](https://blog.expo.dev/expo-sdk-47-a0f6f5c038af)

**[Expo SDK 47](https://blog.expo.dev/expo-sdk-47-a0f6f5c038af)**

新版 SDK 47 已正式發布，並附帶一篇更詳細的新功能介紹文章。

- React Native 0.70.5 和 React 18.1.0
- Expo Modules API 1.0 讓您能夠非常輕鬆地在 Swift/Kotlin 中建立原生的 Fabric 模組
- Hermes 已在 Expo Go iOS 上可用，簡化了除錯體驗。Hermes 將在下一版 SDK 中成為預設引擎
- 多個 Expo 模組支援 Fabric
- Expo Router 測試版：用於 React-Navigation 的檔案系統路由
- 各種 EAS 改進：端對端安全性、M1 工作程序...

總而言之，這是一個非常棒的版本！透過 Expo Modules，採用新架構似乎比官方工具（如 react-native-codegen，目前還不支援 Swift）更容易。
