## ⚛️ React {#react}

[![useSignal() is the Future of Web Frameworks](/emails/issues/135/signals.jpg)](https://www.builder.io/blog/usesignal-is-the-future-of-web-frameworks)

**[`useSignal()` 是 Web 框架的未來](https://www.builder.io/blog/usesignal-is-the-future-of-web-frameworks)**

Misko（Qwik 和 AngularJS 的創建者）強調了 signals 的優勢，並與 React 的運作方式進行對比。React 直接使用狀態值，而 signals 使用 getter 方法，並在調用時建立訂閱關係。

Signals 能自動實現細粒度的重新渲染優化，無需記憶化處理。僅負責屬性傳遞的中介元件無需重新渲染。

這篇文章發表於 [Angular 宣布](https://twitter.com/angular/status/1625939902046117890)計劃採用 signals 的隔天（現已有[演示](https://twitter.com/sarah_edo/status/1628065696247857152)）。

Andrew Clark 已[評論](https://twitter.com/acdlite/status/1626590880126889984)在 React 中引入 signals 的潛在可能。他目前傾向押注 [React Forget](https://www.youtube.com/watch?v=lGEMwh32soc) 編譯器，預計將透過 React Labs 文章發布進展。我認為 React 逆勢而行並持續[探索現有方向](https://twitter.com/acdlite/status/1625599973420154881)是件好事。

[![Announcing Sandpack 2.0 and a Node.js runtime for any browser](/emails/issues/135/sandpack2.jpg)](https://codesandbox.io/blog/announcing-sandpack-2)

**[宣佈 Sandpack 2.0 及適用於任何瀏覽器的 Node.js 運行環境](https://codesandbox.io/blog/announcing-sandpack-2)**

CodeSandbox 宣佈了 Sandpack 的新版本，這是一個能在瀏覽器中創建互動式即時編碼體驗的工具（程式碼編輯器 + 預覽）。Sandpack 透過專用套件與 React 完美整合。v1 版本已被用於新版 React beta 文件網站、Josh Comeau 及其他許多場景。

此 v2 版本基於全新的 NodeBox 技術，可在瀏覽器中運行 Node.js。能夠運行 Next.js、Vite、Astro 等技術棧。

這讓人聯想到 StackBlitz 的技術，該平台剛[宣佈了 WebContainers API](https://blog.stackblitz.com/posts/webcontainer-api-is-here/)。Sandpack 文章解釋了 NodeBox 與 WebContainers 技術的差異。
