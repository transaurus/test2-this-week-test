## 📱 React-Native {#react-native}

[![React Native 0.77](/emails/issues/218/react-native-0.77.jpg)](https://reactnative.dev/blog/2025/01/21/version-0.77)

[**React Native 0.77**](https://reactnative.dev/blog/2025/01/21/version-0.77)

正如規劃，新版 React Native 正式發布。主要更新包括：

- CSS 支援強化：`display: contents`、`boxSizing`、`mixBlendMode` 及 `outline`
- 支援 Android 15 邊到邊顯示及 16KB 記憶體分頁大小（效能優化）
- iOS 初始化模板採用 Swift AppDelegate

Expo 用戶的好消息：[SDK 52 已支援 v0.77](https://expo.dev/changelog/2025/01-21-react-native-0.77)，同時兼容多項基礎套件（Screens、RNGH、Reanimated）。React 19 支援需等待 v0.78，但可能比預期更早到來！

[![Reanimated 4.0 beta](/emails/issues/218/reanimated4.jpg)](https://blog.swmansion.com/reanimated-4-is-new-but-also-very-familiar-b926dd59aa40)

[**Reanimated 4.0 beta**](https://blog.swmansion.com/reanimated-4-is-new-but-also-very-familiar-b926dd59aa40)

Reanimated 4 進入測試階段，這是重大更新！🔥 動畫核心全面重構。現在您可直接使用常規 CSS 在原生線程實現動畫，歸功於新增支援：

- CSS 屬性轉換（property transitions）
- CSS 關鍵幀動畫（keyframe animations）

這項特性大幅簡化 React Native 微互動的樣板程式碼，並實現跨平台動畫交付。現有 worklet API 保持向後兼容，仍適用於進階需求，但新架構（New Architecture）現為必要條件。

參閱 [Reactiive 將 Reanimated v2 動畫遷移至 v4](https://reactiive.io/articles/css-animations-react-native) 案例，遷移過程大幅簡化。更別提遷移舊版 Reanimated v1 程式碼的挑戰了 🙈。

補充說明：您是否注意到正在積極開發的[實驗性 React `<ViewTransition>` 元件](https://motion.dev/blog/reacts-experimental-view-transition-api)？它未包含在 React-DOM 中，背後可能有令人期待的緣由 👀。
