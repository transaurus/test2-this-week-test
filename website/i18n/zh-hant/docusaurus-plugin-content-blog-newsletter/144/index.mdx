## ⚛️ React {#react}

[![Introducing storage on Vercel](/emails/issues/144/vercel.jpg)](https://vercel.com/blog/vercel-storage)

**[Vercel 儲存服務正式推出](https://vercel.com/blog/vercel-storage)**

Vercel 剛發布其優先考慮邊緣運算的儲存服務，特別適用於 React Server Components 和 Next.js：

- Vercel KV：基於 Upstash 的鍵值儲存 Redis
- Vercel Postgres：基於 Neon 的 PostgreSQL 資料庫
- Vercel Blob：基於 Cloudflare R2 的檔案儲存

正值 [Vercel Ship 週](https://vercel.com/ship)：未來幾天應有更多消息公布。選用圖片暗示明日將發布 Next.js 版本，推出新功能 **Next.js 伺服器操作**。這也揭示了提升 React 應用[漸進增強](https://twitter.com/sebastienlorber/status/1653701538336915456)能力的意圖，方式類似 Remix：此表單可在無 JavaScript 或水合作用情況下運作。

注意：此範例不易受到 SQL 注入攻擊，因為它使用了範本字面值（template literals）。

[![Million.js - Virtual DOM: Back in Block](/emails/issues/144/million.jpg)](https://millionjs.org/blog/virtual-dom)

**[Million.js - Virtual DOM：重返區塊](https://millionjs.org/blog/virtual-dom)**

互動式文章介紹 Million.js 及其「區塊虛擬 DOM」概念。其差異比對方法與眾不同：採用非遞迴方式，基於靜態分析、編譯和髒檢查機制，類似 Svelte 的運作原理。

Million.js 並非取代 React，而是與其整合以提升效能。建議僅在特定場景使用：當應用包含大量靜態內容且 UI 樹結構穩定時。

## 📱 React-Native {#react-native}

[![介紹 React Native macOS 0.71](/emails/issues/144/rnmacos.jpg)](https://devblogs.microsoft.com/react-native/2023-04-27-announcing-macos-71/)

**[介紹 React Native macOS 0.71](https://devblogs.microsoft.com/react-native/2023-04-27-announcing-macos-71/)**

Microsoft 跳過了 v0.69 和 v0.70 版本，直接發布 v0.71。現在的目標是與其他平台保持同步：v0.72 的工作正在進行中。這種平台版本對齊看起來非常有用：根據回饋，在單一 monorepo 中混合不同 React-Native 版本並不容易。

新發布的 v0.71 帶來了對 Fabric 的實驗性支援，以及最近在其他平台上發布的所有新功能：TypeScript、Flexbox...。Microsoft 和 Meta 合作，此平台在兩家公司內的使用似乎正在增長：例如，Meta 在 Messenger Desktop 上使用 React-Native macOS。
